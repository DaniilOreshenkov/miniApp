# Инструкция: подключение боевой ЮКассы

## 1. Личный кабинет ЮКассы

### 1.1 Получить боевые ключи
1. Зайди на [yookassa.ru](https://yookassa.ru) → **Интеграция** → **Ключи API**
2. Скопируй:
   - `Идентификатор магазина (shopId)` — числовой ID
   - `Секретный ключ` — начинается с `live_` (не `test_`)

### 1.2 Настроить webhook
1. **Интеграция** → **HTTP-уведомления**
2. Нажми **Добавить webhook**
3. URL: `https://<твой-домен>.vercel.app/api/webhook`
4. Событие: ✅ `payment.succeeded`
5. Сохрани

### 1.3 Включить сохранение платёжных методов
1. **Интеграция** → **Настройки**
2. Убедись что **"Сохранение платёжных данных"** включено (нужно для автосписания)

---

## 2. Vercel — переменные окружения

Зайди: **Vercel** → твой проект → **Settings** → **Environment Variables**

### Добавить / проверить:

| Переменная | Значение | Окружение |
|---|---|---|
| `YOOKASSA_SHOP_ID` | числовой ID магазина | Production |
| `YOOKASSA_SECRET_KEY` | `live_...` ключ | Production |
| `KV_REST_API_URL` | уже стоит ✅ | All |
| `KV_REST_API_TOKEN` | уже стоит ✅ | All |
| `APP_URL` | `https://<твой-домен>.vercel.app` | All |
| `CRON_SECRET` | случайный токен (см. ниже) | All |

### Как сгенерировать CRON_SECRET
Открой терминал и выполни:
```bash
openssl rand -hex 32
```
Скопируй результат и вставь как значение `CRON_SECRET`.

---

## 3. Redeploy

После добавления переменных — **обязательно передеплой**:

**Vercel** → **Deployments** → последний деплой → **⋯** → **Redeploy**

Без этого новые переменные не применятся.

---

## 4. Проверка что всё работает

### Шаг 1 — тестовая покупка
1. Открой приложение в Telegram
2. Купи план **Стартер (169₽)**
3. Пройди оплату до конца

### Шаг 2 — проверить webhook
1. Зайди: **Vercel** → **Logs** → фильтр `/api/webhook`
2. Должна быть запись с `payment.succeeded`
3. Лог должен содержать: `plan saved for <userId>`

### Шаг 3 — проверить активацию плана
Открой в браузере:
```
https://<твой-домен>.vercel.app/api/check-plan?userId=<твой-telegram-id>
```
Ответ должен быть:
```json
{ "planId": "starter" }
```

### Шаг 4 — проверить крон (только для monthly/pro)
1. **Vercel** → **Settings** → **Crons**
2. Убедись что `/api/cron` показан и активен
3. Расписание: `0 9 * * *` (каждый день в 09:00 UTC)

## 5. Возможные проблемы

| Симптом | Причина | Решение |
|---|---|---|
| `confirmationUrl` не приходит | Неверный `YOOKASSA_SECRET_KEY` | Проверь что ключ `live_`, не `test_` |
| Webhook не срабатывает | Не добавлен URL в ЮКасса | Повтори шаг 1.2 |
| План не активируется после оплаты | Webhook получает ошибку | Проверь Vercel Logs → `/api/webhook` |
| Автосписание не работает | Нет `APP_URL` или `CRON_SECRET` | Добавь переменные и передеплой |
| Ошибка 401 в кроне | `CRON_SECRET` не совпадает | Убедись что одно и то же значение везде |
